🔥 Firebase Firestore Database 생성 확인

현재 문제: 400 (Bad Request) 에러 발생
원인: Firestore Database가 생성되지 않음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 해결 방법:

1. Firebase Console 접속
   https://console.firebase.google.com/

2. vibe-code-todo 프로젝트 선택

3. 왼쪽 메뉴 → "Firestore Database" 클릭

4-A. "데이터베이스 만들기" 버튼이 보이면:
   → 클릭
   → "테스트 모드로 시작" 선택
   → 다음
   → 위치: asia-northeast3 (Seoul) 선택
   → 사용 설정
   → 완료될 때까지 대기 (30초~1분)

4-B. 이미 "데이터" 탭이 보이면:
   → "규칙(Rules)" 탭 클릭
   → 아래 코드로 교체:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

   → "게시(Publish)" 버튼 클릭

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ 중요 체크리스트:

□ Firebase Console에서 Firestore Database 메뉴를 클릭했나요?
□ "데이터베이스 만들기" 버튼을 클릭했나요?
□ 데이터베이스 생성이 완료되었나요? (프로비저닝 완료)
□ 보안 규칙을 수정하고 "게시(Publish)"를 클릭했나요?
□ 브라우저를 완전히 닫고 다시 열었나요?

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2. F12 → Console 탭
3. "Firebase 데이터 수신: 0 개" 메시지 확인
4. 할일 추가 테스트
